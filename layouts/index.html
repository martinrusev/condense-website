Automatically apply your CAPS(Chef, Ansible, Puppet and Salt) provisioning scripts. Condense will run them in parallel and report execution stats in real time. You can apply provisioning scripts on build, schedule, retry, receive notifications on failure, etc.

Frequently Asked Questions

  1. What is Condense?

    Condense is a service modelling tool by Martin Rusev, the creator of Amon. Condense allows you to model, configure, provision, maintain, deploy and scale cloud services quickly and efficiently on public clouds.

  2. How does Condense work?

    Condense is a GUI build on top of Terraform by Hashicorp. Behind the scenes Condense generates and applies Terraform states.

  3. Can I use Condense with Puppet, Chef, Ansible or SaltStack?

    Condense works with all CAPS tools out of the box. You can mix and match different provisioning scripts - for example you can deploy MySQL with an Ansible playbook and configure security settings with a Puppet manifest. Condense works similarly to Ansible Tower and Puppet Enterpise - you can apply provisioning scripts immediately after the build phase, automatically retry on failure, schedule execution for maintanance related tasks, receive notifications, etc.

  4. What Distros/Operating systems are supported

    The Condense client is a single static Golang binary and can run on any Windows or Linux machine.
