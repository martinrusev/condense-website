Condense is a cloud orchestration tool that enables you to quickly model, deploy and manage your whole infrastructure

Build

Model your cloud services with a few clicks. An intuitive, universal interface that works across Amazon Web Services, DigitalOcean, Google Compute Cloud and Microsoft Azure.

Provision

Automatically apply your CAPS(Chef, Ansible, Puppet and Salt) provisioning scripts. Condense will run them in parallel and report execution stats in real time. You can apply provisioning scripts on build, schedule, retry, receive notifications on failure, etc.

Frequently Asked Questions

  1. What is Condense?

    Condense is a service modelling tool by Martin Rusev, the creator of Amon. Condense allows you to model, configure, provision, maintain, deploy and scale cloud services quickly and efficiently on public clouds.

  2. How does Condense work?

    Condense is a GUI/web app build on top of Terraform by Hashicorp. Behind the scenes Condense generates and applies Terraform states.

    Condense validates the state files against existing cloud provider APIs and displays all available options.

    Condense stores your infrastructure data in MongoDB GridFS files - you have access to detailed audit data and all changes are versioned.

  3. Can I use Condense with Puppet, Chef, Ansible or SaltStack?

    Condense works with all CAPS tools out of the box. You can mix and match different provisioning scripts - for example you can deploy MySQL with an Ansible playbook and then configure your server with a Puppet manifest. Condense works similarly to Ansible Tower and Puppet Enterprise - you can apply provisioning scripts immediately after the build phase, automatically retry on failure, schedule execution for maintenance related tasks, receive notifications, etc.

  4. What Distros/Operating systems are supported

    The Condense client is a single static Golang binary and can run on any Windows or Linux machine.

  5. In summary, why should I use Condense?

    Condense is the fastest way to quickly model and deploy infrastructure across different cloud providers.

    Works with your existing configuration management tools.

    Provisioning portability: Save multiple provisioning scripts as a bundle and deploy on your laptop, Amazon EC2 or any other cloud provider.
